# 🏗️ Архитектура системы фильтрации возвратов платежей

## 📋 Обзор проекта

Система автоматической фильтрации банковских выписок для выявления возвратов платежей клиентам. Система принимает Excel-файлы из разных банков (Сбер, Тинькофф, ВТБ, etc.), применяет три критерия фильтрации и генерирует подробный отчет.

---

## 🎯 Три критерия фильтрации (AND логика - все должны совпадать)

| № | Критерий | Пример |
|---|----------|--------|
| 1️⃣ | **Сумма кратна 5000 РУБ** | 5000, 10000, 15000, 20000, 123000 |
| 2️⃣ | **Email в столбце "Назначение платежа"** | `user@example.com` в тексте |
| 3️⃣ | **Текст о возврате** | "Возврат оплат по договору", "Возврат средств", "Refund" |

**Важно**: Все 3 условия должны быть истинны одновременно для включения строки в отчет.

## 📦 Основные компоненты

### 1. **FILE HANDLER** (`api/file_handler.py`)
```
Задача: Управлять загруженными файлами

ЧТО:
  • Принимать Excel файлы из POST запроса
  • Валидировать расширение (.xlsx, .xls)
  • Сохранять во временную папку
  • Определять кодировку и формат

ПОЧЕМУ:
  • Безопасность: проверка типа файла
  • Масштабируемость: поддержка разных форматов
  • Надежность: обработка ошибок загрузки

КАК:
  from openpyxl import load_workbook  # Чтение Excel
  from pathlib import Path
  
  1. Получить файл из request.files
  2. Проверить расширение
  3. Сохранить в temp папку
  4. Вернуть путь для обработки
```

### 2. **BANK PARSER FACTORY** (`core/bank_parsers/parser_factory.py`)
```
Задача: Определить тип банка и выбрать нужный парсер

ЧТО:
  • Анализировать название файла (Sber, Tinkoff, VTB)
  • Анализировать структуру Excel (названия столбцов)
  • Вернуть подходящий парсер

ПОЧЕМУ:
  • Разные банки используют разные форматы
  • Сбербанк: "Дата", "Сумма", "Назначение платежа"
  • Тинькофф: "Дата операции", "Сумма операции", "Описание"
  • VTB: свои названия столбцов

КАК:
  def detect_bank(filename, df_sample):
      if 'sber' in filename.lower():
          return SberParser()
      elif 'tinkoff' in filename.lower():
          return TinkoffParser()
      elif 'vtb' in filename.lower():
          return VTBParser()
      else:
          return GenericParser()
```

### 3. **BANK-SPECIFIC PARSERS** (`core/bank_parsers/`)
```
Задача: Преобразовать разные форматы в единый DataFrame

ЧТО:
  • Читать Excel файл
  • Найти нужные столбцы (может быть разные имена)
  • Нормализовать данные
  • Вернуть DataFrame с единообразной структурой

НАПРИМЕР - SberParser:
  
  Входной Excel от Сбера:
  ┌──────────┬────────┬────────────────────┬────────┐
  │ Дата     │ Сумма  │ Назначение платежа │ Статус │
  ├──────────┼────────┼────────────────────┼────────┤
  │ 01.04.26 │ 5000   │ user@mail.ru       │   OK   │
  └──────────┴────────┴────────────────────┴────────┘
  
  На выходе нормализованный DataFrame:
  ┌────────────┬─────────┬──────────────────────┐
  │ date       │ amount  │ payment_purpose      │
  ├────────────┼─────────┼──────────────────────┤
  │ 2026-04-01 │ 5000.0  │ user@mail.ru         │
  └────────────┴─────────┴──────────────────────┘

ПОЧЕМУ разные парсеры:
  • Каждый банк использует разные названия столбцов
  • Разные форматы дат
  • Разные символы для разделения тысяч
  • Упрощает добавление новых банков
```

### 4. **REFUND DETECTOR** (`core/refund_detector.py`)
```
Задача: Применить три критерия фильтрации

ФИЛЬТР 1 - AmountFilter:
┌─────────────────────────────────────┐
│ Проверка: amount % 5000 == 0        │
├─────────────────────────────────────┤
│ Входные суммы: 5000, 10001, 15000   │
│ Результат:                          │
│   ✅ 5000   (5000 % 5000 = 0)      │
│   ❌ 10001  (10001 % 5000 = 1)     │
│   ✅ 15000  (15000 % 5000 = 0)     │
└─────────────────────────────────────┘

ФИЛЬТР 2 - EmailFilter:
┌──────────────────────────────────────────┐
│ Проверка: regex в "Назначение платежа"  │
├──────────────────────────────────────────┤
│ Паттерн: r'[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+' │
│ Текст: "Возврат user@mail.ru за заказ"  │
│ Результат: ✅ НАЙДЕН email               │
└──────────────────────────────────────────┘

ФИЛЬТР 3 - RefundReasonFilter:
┌────────────────────────────────────────┐
│ Проверка: Ключевые слова в назначении │
├────────────────────────────────────────┤
│ Ключевые слова:                        │
│   • "возврат"                          │
│   • "refund"                           │
│   • "возврат оплат по договору"        │
│   • "return"                           │
│                                        │
│ Примеры:                               │
│ ✅ "Возврат оплат по договору"        │
│ ✅ "REFUND for order 12345"            │
│ ❌ "Оплата по счету 12345"             │
└────────────────────────────────────────┘

КАК работает:
  df_filtered = df.copy()
  df_filtered = amount_filter.apply(df_filtered)      # Первый фильтр
  df_filtered = email_filter.apply(df_filtered)       # Второй фильтр
  df_filtered = refund_reason_filter.apply(df_filtered) # Третий фильтр
  
  # Теперь в df_filtered только строки, которые прошли все 3 фильтра
```

  ┌─────────┬──────────────────────────────┐

## 🛡️ Обработка ошибок

```
┌─────────────────────────────────┐
│ Возможные ошибки и обработка    │
├─────────────────────────────────┤
│ ❌ Неверный формат файла        │
│    → Ошибка 400: "Invalid file" │
│                                 │
│ ❌ Не распознан тип банка        │
│    → Использовать GenericParser │
│                                 │
│ ❌ Отсутствуют нужные столбцы  │
│    → Ошибка 422: "Unprocessable"│
│                                 │
│ ❌ Нет совпадений по фильтрам   │
│    → Вернуть пустой отчет       │
│                                 │
│ ❌ Ошибка при генерации Excel   │
│    → Fallback на CSV             │
└─────────────────────────────────┘
```

## ✨ Почему эта архитектура лучше всего

| Аспект | Преимущество |
|--------|-------------|
| **Модульность** | Легко добавить новый банк - создать 1 класс |
| **Тестируемость** | Каждый компонент можно тестировать отдельно |
| **Масштабируемость** | Легко обработать 100 файлов одновременно |
| **Гибкость** | Легко добавить новый фильтр (4-й, 5-й и т.д.) |
| **Надежность** | Валидация на каждом этапе |
| **Производительность** | Pandas для быстрой обработки больших данных |
| **Понятность** | Каждый компонент отвечает за одну задачу |

## 🚀 Следующие шаги реализации

1. **Создать основные модели** (`models.py`)
   - Request/Response схемы
   - DataFrame схемы

2. **Реализовать базовые парсеры** (`core/bank_parsers/`)
   - BaseParser (abstract)
   - SberParser (для вашего примера)
   - GenericParser (fallback)

3. **Реализовать фильтры** (`core/filters/`)
   - AmountFilter
   - EmailFilter
   - RefundReasonFilter

4. **Создать RefundDetector** (`core/refund_detector.py`)
   - Объединить фильтры
   - Собрать статистику

5. **Реализовать API** (`api/routes.py`)
   - POST /upload
   - POST /analyze
   - GET /export

6. **Создать UI** (`frontend/`)
   - Форма загрузки файлов
   - Отображение результатов

## 📝 Пример использования API

```bash
# 1. Загрузить файл
curl -X POST http://localhost:8000/upload \
  -F "files=@statement.xlsx"

# Ответ:
{
  "file_id": "abc123",
  "filename": "Sberbank_April_2026.xlsx",
  "detected_bank": "sber"
}

# 2. Анализировать файл
curl -X POST http://localhost:8000/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "file_id": "abc123",
    "filters": {
      "amount_multiple": 5000,
      "extract_email": true,
      "refund_keywords": ["возврат", "refund"]
    }
  }'

# Ответ:
{
  "total_refund_amount": 123000,
  "refund_count": 24,
  "unique_customers": ["user1@mail.ru", "user2@mail.ru"],
  "transactions": [...]
}

# 3. Экспортировать отчет
curl -X GET "http://localhost:8000/export/abc123?format=excel" \
  --output report.xlsx
```
