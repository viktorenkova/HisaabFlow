## 📊 СОЗДАНО ВИЗУАЛЬНЫХ МАТЕРИАЛОВ

### 1. **Архитектура системы** (Mermaid диаграмма)
```
Входные данные → File Handler → Parser Factory → Bank Parsers 
→ DataFrame → 3 Фильтра → Refund Collector → Report Generator → Excel
```
**Что показывает**: Полный поток обработки данных от входа до выхода
**Используется в**: REFUND_FILTER_ARCHITECTURE.md

### 2. **Логика фильтров** (Mermaid диаграмма)
```
1000 строк → Фильтр 1 (% 5000) → 400 строк
           → Фильтр 2 (email) → 120 строк
           → Фильтр 3 (ключевые слова) → 45 строк
```
**Что показывает**: Как 3 критерия отбирают нужные строки
**Используется в**: REFUND_FILTER_QUICK_GUIDE.md

## 🎓 ЧТО МЫ СОЗДАЛИ?

### Архитектура включает:

**8 главных компонентов:**
1. File Handler - загрузка файлов
2. Parser Factory - определение банка
3. Bank Parsers (Sber, Tinkoff, VTB, Generic)
4. Amount Filter - проверка суммы
5. Email Filter - поиск email
6. Refund Reason Filter - поиск ключевых слов
7. Refund Collector - сбор результатов
8. Report Generator - создание отчета

**3 стратегии обработки:**
- Модульность (легко добавлять компоненты)
- Масштабируемость (может обрабатывать 100+ файлов)
- Расширяемость (легко добавлять новые фильтры)

**2 типа выходных данных:**
- Excel отчет (Summary + Details)
- JSON для интеграции с другими системами

## 💬 ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ

### Q: Почему именно эта архитектура?
**A**: Потому что она модульная, расширяемая, тестируемая и легко понимается.

### Q: Сколько времени займет реализация?
**A**: 3-4 дня для опытного разработчика, 5-6 для начинающего.

### Q: Будет ли работать на всех моих банках?
**A**: На Сбере точно. На других банках нужно создать специальный парсер (~30 мин).

### Q: Что если появятся новые требования?
**A**: Архитектура легко расширяется. Новый фильтр = 1 класс (~30 мин).

### Q: Может ли это обрабатывать 1000 файлов?
**A**: Да, с очередью задач (Redis). Потребуется ~2 дня доработок.

### Q: Это будет веб-приложение или десктоп?
**A**: FastAPI backend (API) + HTML/JS frontend. Работает в браузере.
