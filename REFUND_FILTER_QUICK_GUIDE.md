# 📌 Ключевые моменты архитектуры - Шпаргалка

## 🎯 Миссия системы в одном предложении
**Автоматически фильтровать банковские выписки, чтобы найти только возвраты платежей клиентам (по 3 критериям) и создать отчет за 10 секунд вместо 30 минут ручной работы.**

---

## ⚙️ Как это работает? (Упрощенно)

```
1. Вы загружаете Excel файл выписки  →  Система загружает файл
2. Система определяет, от какого банка  →  Выбирает правильный "переводчик"
3. "Переводчик" преобразует в стандартный формат  →  Все банки теперь одинаково
4. Применяются 3 фильтра ПО ОЧЕРЕДИ  →  Отбираются только нужные строки
5. Собирается статистика и отчет  →  Excel с результатами
```

---

## 🔑 5 главных компонентов

### 1️⃣ **FILE HANDLER** - Охранник
- **Что делает**: Проверяет, что загружен именно Excel файл
- **Почему важно**: Безопасность (не загрузить вирус)
- **Пример**: `statement.xlsx` ✅ | `statement.txt` ❌

### 2️⃣ **BANK PARSER FACTORY** - Детектив  
- **Что делает**: Определяет "откуда" файл (Сбер? Тинькофф? ВТБ?)
- **Почему важно**: Разные банки используют разные названия столбцов
- **Пример**: 
  - Сбер использует "Дата"
  - Тинькофф использует "Дата операции"

### 3️⃣ **BANK PARSERS** - Переводчики
- **Что делают**: Берут Excel от конкретного банка и переводят в единый формат
- **Почему важно**: После парсинга все выглядят одинаково
- **Пример**:
  ```
  INPUT (Сбер):          OUTPUT (Unified):
  "Дата" → "01.04.26"    "date" → "2026-04-01"
  "Сумма" → "5000.5"     "amount" → 5000.5
  "Назначение" → "text"  "payment_purpose" → "text"
  ```

### 4️⃣ **FILTERS** - Охранники трех врат
- **Фильтр 1**: Сумма должна быть кратна 5000
  - `5000 % 5000 = 0` ✅ Пропускаем
  - `5001 % 5000 = 1` ❌ Отклоняем
  
- **Фильтр 2**: В "Назначение платежа" должен быть email
  - Ищем паттерн: `something@something.com`
  
- **Фильтр 3**: В "Назначение платежа" должны быть ключевые слова
  - Ищем: "Возврат", "Refund", "Возврат оплат по договору"

### 5️⃣ **REPORT GENERATOR** - Секретарь
- **Что делает**: Создает красивый отчет Excel
- **Что в отчете**:
  - Лист 1: Summary (итоги, статистика)
  - Лист 2: Details (полный список всех совпадений)

## 🔄 Жизненный цикл запроса

```
┌─ Пользователь загружает файл
│
├─ FILE HANDLER
│  └─ Проверка: это Excel?
│     └─ Сохраняем во временную папку
│
├─ PARSER FACTORY  
│  └─ Анализируем имя: "Sberbank_Apr2026.xlsx"
│     └─ Определено: SberParser
│
├─ SBER PARSER
│  └─ Читаем Excel
│     └─ Ищем колонки: "Дата", "Сумма", "Назначение платежа"
│     └─ Нормализуем формат (даты, цифры)
│     └─ Результат: DataFrame с 1000 строк
│
├─ FILTER 1: AMOUNT
│  └─ Проверяем: amount % 5000 == 0
│     └─ Результат: 400 строк прошли
│
├─ FILTER 2: EMAIL
│  └─ Ищем: regex '@' в "Назначение платежа"
│     └─ Результат: 120 строк прошли
│
├─ FILTER 3: REFUND KEYWORDS
│  └─ Ищем: "возврат" или "refund"
│     └─ Результат: 45 строк прошли
│
├─ REPORT GENERATOR
│  ├─ Summary: 45 возвратов, 223000 РУБ
│  ├─ Уникальные emails: 28 шт.
│  └─ Details: Полный список 45 транзакций
│
└─ EXPORT
   └─ Создаем report.xlsx
      └─ Пользователь скачивает

Время выполнения: ~2-5 секунд вместо 30 минут ✨
```

## 💡 Почему именно эта архитектура?

### ✅ Модульность
- Хотите добавить поддержку новому банку?
- Просто создайте класс `YourBankParser` и добавьте в Factory
- Остальной код не меняется

### ✅ Расширяемость  
- Хотите добавить 4-й критерий фильтра?
- Создайте класс `NewFilter` наследуя `BaseFilter`
- Добавьте в RefundDetector
- Остальной код не меняется

### ✅ Тестируемость
- Каждый компонент можно тестировать отдельно
- Нет сложных зависимостей
- Можно мокировать любую часть

### ✅ Понятность
- Каждый класс делает одну работу
- Названия понятны на русском
- Комментарии объясняют "почему"

### ✅ Производительность
- Pandas для быстрой обработки таблиц
- Параллельная обработка нескольких файлов
- Можно обработать 100 файлов за раз

## 🚨 Граничные случаи и обработка ошибок

```
ПРОБЛЕМА                      РЕШЕНИЕ
───────────────────────────   ──────────────────────
Неверный формат файла         ❌ Ошибка 400
Не распознан тип банка        ✅ Используем GenericParser
Отсутствуют нужные столбцы   ❌ Ошибка 422
Нет совпадений по фильтрам   ✅ Пустой отчет (0 результатов)
Битый Excel файл              ❌ Ошибка при парсинге
Email не найден в тексте      ✅ Просто отклоняем строку
Очень большой файл (10 МБ)   ✅ Обработаем, но медленнее
```

## 📱 API Endpoints (будущие)

```
POST /upload
  Загрузить Excel файл(ы)
  Ответ: file_id, detected_bank, preview

POST /analyze
  Запустить фильтрацию
  Параметры: file_id, фильтры (опционально)
  Ответ: результаты анализа

GET /export/{file_id}
  Скачать готовый отчет Excel
  Параметры: format (excel/csv)
  Ответ: файл
```
