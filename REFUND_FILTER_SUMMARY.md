## 🏗️ АРХИТЕКТУРА В ДВУХ СЛОВАХ

```
ВХОДНЫЕ ДАННЫЕ (Excel)
        ↓
ОПРЕДЕЛЕНИЕ ТИПА БАНКА (Factory)
        ↓
НОРМАЛИЗАЦИЯ ДАННЫХ (Parser)
        ↓
ПРИМЕНЕНИЕ 3 ФИЛЬТРОВ (Filters 1, 2, 3)
        ↓
СБОР СТАТИСТИКИ (Refund Collector)
        ↓
СОЗДАНИЕ ОТЧЕТА (Report Generator)
        ↓
ЭКСПОРТ (Excel/CSV)
        ↓
ВЫХОДНЫЕ ДАННЫЕ (Готовый отчет)
```

## 🔑 КЛЮЧЕВЫЕ КОМПОНЕНТЫ

| № | Компонент | Задача | Входные данные | Выходные данные |
|---|-----------|--------|---|---|
| 1️⃣ | **File Handler** | Загрузка & валидация файлов | Excel файл | Сохраненный файл + путь |
| 2️⃣ | **Parser Factory** | Определение типа банка | Имя файла + данные | Выбранный парсер |
| 3️⃣ | **Bank Parsers** | Нормализация формата | Raw Excel данные | Единообразный DataFrame |
| 4️⃣ | **Amount Filter** | Проверка: сумма % 5000 | DataFrame | Отфильтрованный DataFrame |
| 5️⃣ | **Email Filter** | Проверка: email найден | DataFrame | Отфильтрованный DataFrame |
| 6️⃣ | **Refund Reason Filter** | Проверка: ключевые слова | DataFrame | Отфильтрованный DataFrame |
| 7️⃣ | **Refund Collector** | Сбор результатов | Отфильтрованный DataFrame | JSON с результатами |
| 8️⃣ | **Report Generator** | Создание отчета | JSON результатов | Excel файл |

## 📐 ТЕХНИЧЕСКИЕ РЕШЕНИЯ

### Язык и фреймворк
- **Python 3.8+** - простой, быстрый, много библиотек
- **FastAPI** - современный веб-фреймворк для API
- **Pandas** - обработка табличных данных
- **openpyxl** - работа с Excel файлами

### Паттерны проектирования
- **Factory Pattern** - для выбора нужного парсера
- **Strategy Pattern** - для фильтров (легко добавлять новые)
- **Pipeline Pattern** - последовательное применение фильтров
- **Abstract Base Classes** - для расширяемости

### Структура данных
- **Unified DataFrame** - все банки в одинаковый формат
- **Pydantic Models** - валидация данных, автодокументация

### ✅ "Что это за система и для чего?"
Система автоматизирует поиск возвратов в банковских выписках.

### ✅ "Когда будет готово?"
2-3 дня на реализацию + 1 день на тестирование = 3-4 дня.
